# Plan: Add Light Theme + Theme Toggle in Top Bar

## Goal
Add a Gruvbox Light theme and a Sun/Moon toggle button in the Header (top bar) to switch between light and dark mode.

## Current State
- All 6 existing themes are dark: gruvbox, midnight, nord, solarized, rose-pine, catppuccin.
- Theme system uses `data-theme` attribute on `<html>`.
- Default (no attribute) = Gruvbox Dark.
- Tokens: `frontend/src/ui/tokens.css` (default dark values).
- Theme overrides: `frontend/src/ui/themes.css`.
- Store: `frontend/src/store/prefsSlice.ts` — `ThemeId` type, `theme`/`setTheme`.
- Theme picker (color dots) lives in `LogsFooter.jsx` bottom bar.
- Header (`frontend/src/components/Header.jsx`) has no theme controls.
- `themes.css` already has empty scaffolding for `[data-theme="auto"]` with `@media (prefers-color-scheme: light)`.

## Changes

### 3. Add theme toggle button in Header (`frontend/src/components/Header.jsx`)

Add a Sun/Moon icon button in the right section of the header (before WaveBars):

- Import `Sun`, `Moon` from `lucide-react`.
- Import `useAppStore` from store.
- Read `theme` and `setTheme` from store.
- Determine if current theme is light: `const isLight = theme === 'gruvbox-light'`.
- On click: toggle between `'gruvbox'` (dark) and `'gruvbox-light'` (light).
- Show `Sun` icon when dark (click to go light), `Moon` icon when light (click to go dark).

Position: in `hq-col-right`, before `<NotificationPanel>`.

```jsx
<button
  type="button"
  className="hq-theme-toggle"
  onClick={() => setTheme(isLight ? 'gruvbox' : 'gruvbox-light')}
  title={isLight ? 'Switch to dark mode' : 'Switch to light mode'}
  aria-label={isLight ? 'Switch to dark mode' : 'Switch to light mode'}
>
  {isLight ? <Moon size={14} /> : <Sun size={14} />}
</button>
```

### 6. Handle edge case in `setTheme` logic

Current logic in `prefsSlice.ts`:
- `gruvbox` → removes `data-theme` attribute (default).
- Others → sets `data-theme`.

`gruvbox-light` should set `data-theme="gruvbox-light"` — this already works with the existing else branch. No change needed.

## Risks / Notes

- Some components may have hardcoded dark colors (inline styles, SVG fills). These won't auto-adapt. A visual pass after implementation will catch these.
- The `color-scheme` CSS property should be set to `light` on `<html>` when gruvbox-light is active so native scrollbars/form controls render in light mode. Add to `setTheme`:
  ```ts
  document.documentElement.style.colorScheme = id === 'gruvbox-light' ? 'light' : 'dark';
  ```
- The Header toggle is a simple dark/light flip. Users who want other dark themes still use the bottom-bar dot picker.
